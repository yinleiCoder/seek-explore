### 认识数据  

#### 数据分类  

**数据是对事物的描述记录**。根据**计量层次**，进一步对数据进行分类。如：黄瓜、番茄、森林、书本，无法计算比较，计量层次低。而1，2，3，4，5，这些数据可以计算比较，那么计量层次就高。

- **定类数据**，按照类别属性进行分类，各类别之间是平等并列关系，这种数据不带数量信息，并且不能在各类别间进行排序，针对这类的主要数值运算，计算每一类别中的项目的频度和频率
- **定序数据**，数据之间可以进行排序、比较优劣，通过将编码排序，可以表示之间的高低差异
- **定距数据**，具有一定单位的实际测量值，数据的精确性比定类数据、定序数据更高，可以计算出各变量之间的实际差距(加、减)
- **定比数据**,可以比较大小，进行加减乘除运算，在定距尺度中0表示数值，在定比尺度中0表示没有。定比数据中存在绝对零点，而定距数据不存在

总结，**定性数据**，一组表示事物性质、规定事物类别的文字表述型数据，包括定类数据、定序数据；**定量数据**，指以数量形式存在着的属性，并因此可以对其进行测量的数据，包括定距数据、定比数据。把这些数据以一定的标准组合起来后，就形成了数据矩阵/二维数据表。  

#### 统计指标  

**体现总体数量特征的概念和数值**，根据数据分析的目的不同，统计指标也会变化。如选择理财产品时，指标有往期业绩、风险系数、年化收益。

- **总量指标**,特定条件下的总规模、总水平或工作总量，是一种最基本的统计指标
- **平均指标(集中趋势指标)**,用一个数字显示其一般水平
- **相对指标**，两个有联系的现象数值相比得到的比率，描述的是相对关系，而不是总体情况。其他指标还有比例、比率、倍数、环比、同比
- **集中趋势指标**,体现数据的一般水平的指标，最快速了解样本数据的概况
  - 最常用**平均值**(具有一定误导性，对异常数据不敏感)
  - **中位数**,按顺序排列，居于中间位置的数，更具代表性
  - **众数**，出现次数最多的数值，反应的是局部特征密集度，可以有多个
- **离散趋势指标**，体现内部差异度的指标
  - **极差**，相距最远的两个点之间的距离，数据内部最大的差异情况
  - **平均差**，一组数据与平均值差异的平均差异，数据项与平均值的差距越大，数据越分散。但在事件驱动型的数据，在样本量较小的时候，容易导致误差
  - **标准差**，更能代表离散程度，标准差越大，风险越高 

#### 分布形态    
把数据变成点线面，**数据图表化后呈现出来的形态**。常见形态有：正太分布、左偏分布、右偏分布。  

#### 数据分析流程
- **目标确定**：解决什么问题？达到什么目的？
    - 针对现有情况分析：描述性分析
    - 基于现状，预测未来情况：预测性分析
- **数据获取**
    - 字段设计：如平均销售额、销售总额、增减幅度，这些是根据基础数据(如订单号、交易日期、交易金额)计算得出
    - 数据提取：销售管理软件，导入导出，互联网企业SQL从数据库提取
- **数据清洗**
    - 异常值：识别、判定、处理
    - 空白值：修正
    - 无效值：平均值填补
    - 重复值：删除
- **数据整理**
    - 格式化：日期的处理、行列格式化
    - 指标计算：基础的计算
- **描述分析**
    - 数据描述：数据的基本情况。如数据总数、时间跨度、数据来源等
    - 指标统计：分析实际情况的数据指标。变化、分布、对比、预测
- **洞察结论**：数据报告的核心，体现数据分析能力
- **撰写报告**
    - 报告背景：业务现状
    - 报告目的：解决什么问题
    - 数据基本情况：数据可信度
    - 可视化图表：数据的可理解程度
    - 策略选择：提出解决方案

#### 行业分析方法  

行业分析就是通过调研、资料手机、结构化分析来描述行业规模、发展、产业链，知道有什么、在干什么、需求如何、盈利如何、企业、产品，深入了解行业方方面面，辅助商业决策。

e.g: 开一家火锅店，行业分析有多少家店、日流水、成本VS收益、选址VS交易量、消费群体等。

行业分析就是：
- **以行业为分析对象**：具有高度相似性、竞争性
- **企业相互作用关系**：竞争、合作、供应商、服务商
- **产业本身发展**：市场规模、需求、增速、未来潜力
- **联系、区域分布**：产业链上下游、国家城市分布  

行业分析的整体流程为：(发展背景、竞争分析、生态分析、趋势预测)
- **确定目的**
- **资料收集**
  - 一手信息（没有转手：内部数据库、从业人员、数据抓取）
  - 二手信息（买第三方数据服务商：数据范围广、量级大、统计维度复杂、获取难度高）
- **结构化分析**
  - 市场规模趋势：总量、细分、预测
    - 艾媒数据中心
    - PEST分析模型
      - 政治：政策法规、监管、处罚
      - 经济：宏观经济、可支配资金、当前经济状况
      - 社会：人口因素、消费心理、生活方式、文化传统
      - 技术：5G发展、个性化推荐、智能手机普及
  - 竞争分析：波特五力法(发展战略、产品属性、商业模式、目标用户、市场数据、运营推广策略)
    - 同业竞争

### (二).解析数据指标体系

#### 以用户的行为分析的数据分析模型

以用户、用户产生的价值为分析维度  
(1).用户生命周期模型
用户从接触到离开产品的过程
- 导入期：引导用户注册
- 成长期：体验行为
- 成熟期：依赖行为，发展为核心用户
- 休眠期：一段时间未登录
- 流失期：长时间未登录

(2).AARRR模型
从产品营销角度，实现用户管理
- ACQUISITION: 获取用户(获客) how do users find you?
- ACTIVATION: 提高用户活跃度(激活) do users have a great first experience?
- RETENTION: 提高用户留存率(留存) do users come back?
- REVENUE: 获取收入(变现) how do you make money?
- REFERRAL: 自传播(传播) do users tell others?  
用户生命周期价值LTV > 用户获取成本CAC+用户经营成本COC

(3).RFM模型(xyz轴)
从消费行为角度，对用户进行分层并采取差异化营销策略
- 最近一次消费Recency
- 消费频率Frequency
- 消费金额Monetary

#### 以偏思维的数据分析模型
应用广泛、容易理解、逻辑思维能力  
(1).5W2H模型(七何分析法)
以提问的方式，快速掌握事件本质
- what: 用户画像是什么
- why: 运营活动、营销方案
- who: 年龄、性别、工作、婚姻
- when: 购买时间、频次、金额
- where: 分布地点、聚集性
- how: 行为轨迹、支付方式
- how much: 消费水平、价位

(2).逻辑树模型(类似数据结构中的判定树)
一步步拆解问题，得出解决方案
- 确保考虑的完整性
- 避免重复思考
- 识别关键问题
- 将大问题分解小问题

(3).A/B测试模型
分组测试，利用数据筛选应用方案

#### 以战略分析的数据分析模型
常用于战略分析、竞争分析、行业、企业

#### RFM模型
衡量当前用户价值和客户潜在价值的重要工具和手段。
- Recency: 距离最近一次交易
- Frequency: 交易频率
- Monetary: 交易金额
- 使用Excel计算R、F、M分值

根据这些指标，划分了这些客户，给用户打标签，进行精细化管理：

| 客户类型 | 最近交易距离当前天数 | 累计单数 | 累计交易金额 | 用户精细化管理                 | 
| --- |------------| --- | --- |-------------------------|
| 重要价值客户 | ↑          | ↑|↑ | RFM都很大，优质客户，需要保持        |
| 重要唤回客户 | ↓          | ↑| ↑| 交易金额和交易次数大，但最近无交易，需要唤回  |
| 重要深耕客户 |  ↑          |↓ |↑ | 交易金额大贡献度高，且最近有交易，需要重点识别 |
| 重要挽留客户 |   ↓         | ↓| ↑| 交易金额大，潜在的有价值客户，需要挽留     |
| 潜力客户 |      ↑      | ↑|  ↓| 交易次数大，且最近有交易，需要挖掘       |
| 新客户 |         ↑   | ↓| ↓| 最近有交易，接触的新客户，有推广价值      |
| 一般维持客户 |      ↓      | ↑| ↓| 交易次数多，但是贡献不大，一般维持       |
| 流失客户 |          ↓  |↓ | ↓| FM低过平均值，最近也没再发货相当于流失    |

(1).设置评分标准
根据获取到的RFM指标建立RFM评分卡，并根据打分卡对数据计算R、F、M得分。这里以用户的订单数:0~100+, 100/5=20为例

| RFM | 业务含义       | 1分 | 2分 | 3分 | 4分 | 5分 |
|-----|------------| --- | --- | --- | --- | --- |
| R   | 最近交易距离当前天数 | <20 | 21~24 | 41~60 | 61~80 | >80 |
| F   | 累计单数       | |  | | | |
| M   | 累计交易金额     | |  | | | |

(2).给用户贴标签  
根据每个用户的RFM得分，对用户进行分类/打标签
- 基于均值贴标签：利用所有用户的R、F、M得分的各均值进行标签区分，再根据计算出的R、F、M标签值去vlookup匹配客户类型
- 均值切分不太合理，一刀切了异常值，所以采用K-Means对RFM评分卡进行数据分组优化
  - K-Means是将数据进行分组的聚类数据算法(python sklearn)
    - 首先将数据分为K组
    - 随机选取K个对象作为初始的聚类中心
    - 计算每个对象与各个聚类中心之间的距离
    - 把每个对象分配给距离它最近的聚类中心
    - 直到没有对象被重新分配给不同的聚类

#### 流量 
流动的数量，一定时间内打开某个网站地址的人气访问量。
流量明星：流量越大、数据越好、商业价值越高。

#### 流量的价值  
流量大，带来关注度高，进而企业变现能力变强。

#### 获取流量
- 付费渠道
- 自身/合作渠道
- 病毒/口碑渠道

#### 流量的数据指标
- 站外营销推广指标：衡量不同渠道的推广效果
  - 曝光量：对用户展示的次数(被加载的次数)
  - 点击量：被用户点击的次数
  - 点击率CTR: =点击量/曝光量。喜爱度、匹配度
  - 每次点击成本CPC：按点击次数付费。体现单位成本支出
  - 每次行动付费CPA：按完成转化目标的数量付费(下载、观看)
  - 每UV成本：=广告费用/UV。实际看到人数
  - 每访问成本：=广告费用/访问量。用户访问频次
  - 投资回报率ROI：=销售额 or 利润 / 费用。核心指标
- 网站流量数量指标：监控目标从站外转移到站内
  - 到达率：=到达量(带站外标记)/点击量.
  - 独立访客UV：一定时间内的访客数量
  - 访问次数Visit：访问人次(超过30分钟)。1个UV多个Visit
  - 页面浏览量PV:站内页面的曝光量。1个Visit多个PV
  - 新访问占比：=新访问量/总访问量。吸引力
- 网站流量质量指标：关注网站对用户的吸引力
  - 访问深度：=PV/访问量。访问内人均页面浏览量，访问质量、吸引力
  - 停留时间：=最后一次请求时间戳 - 第一次请求时间戳
  - 跳出/跳出率：没有点击第二个页面即离开。=跳出访问/落地页访问。体现用户第一反应
  - 退出/退出率：从网站上离开的行为，=页面退出的访问/该页面访问。越低越好，工具类
  - 产品页转化率：=产品页访问量/总访问量。订单转化重要指标
  - 加入购物车转化率：=加入购物车访问量/总访问量。用户购物意向
  - 结算转化率：=结算访问量/总访问量。用户购物意向
  - 下载转化率：=下载访问量/总访问量。用户目标性
  - 注册转化率：=注册会员量/总UV数
  - 购物车内转化率：=提交订单的访问量/加入购物车的访问量。重要监控指标。

#### 企业3大流量分析模型
